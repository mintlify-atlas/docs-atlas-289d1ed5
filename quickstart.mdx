## Prerequisites

Before you begin, make sure you have:

- **Docker** installed on your machine ([Get Docker](https://docs.docker.com/get-docker/))
- A terminal or command prompt
- Basic familiarity with REST APIs and curl commands

<Note>
  Docker is the recommended way to run MarkItDown API as it includes all dependencies (Python, libmagic, Pandoc).
</Note>

## Step 1: Run the API with Docker

<Steps>
  <Step title="Clone or download the repository">
    First, get the MarkItDown API source code:

    ```bash
    git clone <repository-url>
    cd markitdown-api
    ```

    <Info>
      If you received the code as a ZIP file, extract it and navigate to the directory instead.
    </Info>
  </Step>

  <Step title="Build the Docker image">
    Build the container image from the Dockerfile:

    ```bash
    docker build -t markitdown-api .
    ```

    This installs all required dependencies:
    - Python 3.12
    - FastAPI and Uvicorn
    - MarkItDown with all format handlers
    - Pandoc (bundled via pypandoc_binary)
    - libmagic for file type detection
  </Step>

  <Step title="Start the API server">
    Run the container and expose port 8000:

    ```bash
    docker run -d -p 8000:8000 --name markitdown markitdown-api
    ```

    The API is now running at `http://localhost:8000`

    <Accordion title="Optional: Add OpenAI API key for AI-enhanced OCR">
      To enable advanced image text extraction with OpenAI GPT-4o-mini:

      ```bash
      docker run -d -p 8000:8000 \
        -e OPENAI_API_KEY="your-api-key-here" \
        --name markitdown markitdown-api
      ```
    </Accordion>
  </Step>

  <Step title="Verify the API is running">
    Check the health endpoint:

    ```bash
    curl http://localhost:8000/health
    ```

    You should see:

    ```json
    {"status":"ok"}
    ```

    <Info>
      If you get a connection error, wait a few seconds for the container to fully start, then try again.
    </Info>
  </Step>
</Steps>

## Step 2: Convert Your First File to Markdown

Let's convert a document to Markdown using the `/upload` endpoint.

<Steps>
  <Step title="Prepare a test file">
    For this example, create a simple text file:

    ```bash
    echo "# Hello World\nThis is a test document." > test.txt
    ```

    Or use any existing PDF, DOCX, PPTX, Excel, or image file you have.
  </Step>

  <Step title="Upload and convert the file">
    Use curl to upload the file:

    ```bash
    curl -X POST http://localhost:8000/upload \
      -F "files=@test.txt" \
      -o output.md
    ```

    The converted Markdown will be saved as `output.md`.

    <CodeGroup>
      ```bash Single File
      curl -X POST http://localhost:8000/upload \
        -F "files=@document.pdf" \
        -o document.md
      ```

      ```bash Multiple Files
      curl -X POST http://localhost:8000/upload \
        -F "files=@file1.docx" \
        -F "files=@file2.pdf" \
        -o converted_files.zip
      ```

      ```bash Excel with Multiple Sheets
      curl -X POST http://localhost:8000/upload \
        -F "files=@spreadsheet.xlsx" \
        -o sheets.zip
      ```
    </CodeGroup>
  </Step>

  <Step title="View the results">
    Open the output file:

    ```bash
    cat output.md
    ```

    You should see clean Markdown extracted from your document.

    <Warning>
      If you uploaded multiple files or an Excel file with multiple sheets, you'll receive a ZIP file. Extract it to see the individual Markdown files:

      ```bash
      unzip converted_files.zip
      ```
    </Warning>
  </Step>
</Steps>

## Step 3: Convert Markdown to DOCX

Now let's go the other direction - convert Markdown to a Word document.

<Steps>
  <Step title="Create a Markdown file">
    Create a sample Markdown file:

    ```bash
    cat > sample.md << 'EOF'
    # Project Proposal

    ## Executive Summary
    This document outlines the key objectives and deliverables.

    ## Timeline
    - **Phase 1**: Research (2 weeks)
    - **Phase 2**: Development (4 weeks)
    - **Phase 3**: Testing (2 weeks)

    ## Budget
    | Category | Amount |
    |----------|--------|
    | Development | $50,000 |
    | Marketing | $20,000 |
    | Total | $70,000 |
    EOF
    ```
  </Step>

  <Step title="Convert to DOCX with default template">
    Use the `/convert` endpoint:

    ```bash
    curl -X POST http://localhost:8000/convert \
      -F "file=@sample.md" \
      -F "format=docx" \
      -F "use_default_template=true" \
      -o proposal.docx
    ```

    <CodeGroup>
      ```bash DOCX with default template
      curl -X POST http://localhost:8000/convert \
        -F "file=@sample.md" \
        -F "format=docx" \
        -F "use_default_template=true" \
        -o output.docx
      ```

      ```bash PPTX presentation
      curl -X POST http://localhost:8000/convert \
        -F "file=@slides.md" \
        -F "format=pptx" \
        -F "use_default_template=true" \
        -o presentation.pptx
      ```

      ```bash HTML standalone document
      curl -X POST http://localhost:8000/convert \
        -F "file=@content.md" \
        -F "format=html" \
        -o page.html
      ```

      ```bash Custom DOCX template
      curl -X POST http://localhost:8000/convert \
        -F "file=@report.md" \
        -F "format=docx" \
        -F "use_default_template=false" \
        -F "template=@custom_template.docx" \
        -o report.docx
      ```
    </CodeGroup>
  </Step>

  <Step title="Open the generated document">
    Open `proposal.docx` in Microsoft Word, Google Docs, or any DOCX-compatible application.

    You should see your Markdown content formatted as a professional Word document with:
    - Proper heading styles
    - Formatted lists
    - Styled tables
  </Step>
</Steps>

## Step 4: Check Available Templates

The API includes default templates for DOCX and PPTX. You can check their availability:

```bash
curl http://localhost:8000/convert/templates
```

Response:

```json
{
  "default_templates": {
    "docx": true,
    "pptx": true
  }
}
```

<Note>
  Default templates are automatically generated by Pandoc on first startup. If they're missing, the API will still work but won't apply template styling.
</Note>

## Advanced: Using Custom Templates

You can provide your own branded DOCX or PPTX template:

<Steps>
  <Step title="Prepare your template">
    Create a DOCX or PPTX file with your desired:
    - Brand colors and fonts
    - Header and footer
    - Logo and styling
    - Default paragraph and heading styles
  </Step>

  <Step title="Upload with custom template">
    Pass the template file along with your Markdown:

    ```bash
    curl -X POST http://localhost:8000/convert \
      -F "file=@content.md" \
      -F "format=docx" \
      -F "use_default_template=false" \
      -F "template=@my_brand_template.docx" \
      -o branded_output.docx
    ```

    <Info>
      The template file must match the output format (`.docx` template for DOCX output, `.pptx` template for PPTX output).
    </Info>
  </Step>
</Steps>

## Common Configurations

<AccordionGroup>
  <Accordion title="Change the upload size limit">
    The default maximum upload size is 50 MB. To increase it:

    ```bash
    docker run -d -p 8000:8000 \
      -e MAX_UPLOAD_MB="100" \
      --name markitdown markitdown-api
    ```
  </Accordion>

  <Accordion title="Configure CORS origins">
    To allow requests from specific origins (for web applications):

    ```bash
    docker run -d -p 8000:8000 \
      -e CORS_ORIGINS="http://localhost:5173,https://myapp.com" \
      --name markitdown markitdown-api
    ```
  </Accordion>

  <Accordion title="View API logs">
    To see what's happening inside the container:

    ```bash
    docker logs -f markitdown
    ```

    Press `Ctrl+C` to stop following logs.
  </Accordion>

  <Accordion title="Stop and remove the container">
    To stop the API:

    ```bash
    docker stop markitdown
    docker rm markitdown
    ```
  </Accordion>
</AccordionGroup>
