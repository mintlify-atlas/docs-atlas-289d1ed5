The MarkItDown API integrates with OpenAI's GPT-4 Vision model to provide advanced optical character recognition (OCR) for images and scanned documents. When enabled, the API can extract text from images, screenshots, and other visual content.

## Overview

OpenAI integration is **optional** and enhances the conversion capabilities:

| Without OpenAI | With OpenAI |
|----------------|-------------|
| Basic text extraction from images | AI-powered OCR with GPT-4 Vision |
| Limited accuracy for complex images | High-accuracy text recognition |
| No context understanding | Contextual understanding of image content |
| Standard MarkItDown processing | Enhanced image description and extraction |

<Info>
The API works perfectly fine without OpenAI. The integration is purely optional and only enhances image processing capabilities.
</Info>

## Configuration

OpenAI integration is controlled by a single environment variable:

```bash
# .env
OPENAI_API_KEY=sk-...
```

The API checks for this configuration:

```python
# app/config.py:7-8
# OpenAI — optional, only needed for AI-enhanced image OCR
OPENAI_API_KEY: str | None = os.getenv("OPENAI_API_KEY")
```

**Source**: `app/config.py:7-8`

<Steps>
  <Step title="Get OpenAI API Key">
    Sign up at [OpenAI](https://platform.openai.com/) and create an API key.
  </Step>

  <Step title="Set Environment Variable">
    Add the key to your `.env` file:
    ```bash
    OPENAI_API_KEY=sk-proj-...
    ```
  </Step>

  <Step title="Restart API Server">
    The configuration is loaded at startup:
    ```bash
    uvicorn app.main:app --reload
    ```
  </Step>

  <Step title="Verify Integration">
    The API will now use OpenAI for image processing automatically.
  </Step>
</Steps>

## How It Works

The converter initialization checks for the OpenAI API key:

```python
# app/services/markitdown_service.py:15-24
def _get_converter() -> MarkItDown:
    """Return a MarkItDown instance. Uses OpenAI for image OCR if API key is set."""
    if OPENAI_API_KEY:
        try:
            import openai
            client = openai.OpenAI(api_key=OPENAI_API_KEY)
            return MarkItDown(mlm_client=client, mlm_model="gpt-4o-mini")
        except Exception:
            pass
    return MarkItDown()
```

**Source**: `app/services/markitdown_service.py:15-24`

### Model Selection

The API uses **GPT-4o-mini** for image processing:

```python
return MarkItDown(mlm_client=client, mlm_model="gpt-4o-mini")
```

<Note>
`gpt-4o-mini` provides an excellent balance of accuracy and cost for OCR tasks. It's optimized for vision tasks while being more economical than full GPT-4.
</Note>

### Fallback Behavior

If OpenAI initialization fails, the API gracefully falls back to standard processing:

```python
try:
    import openai
    client = openai.OpenAI(api_key=OPENAI_API_KEY)
    return MarkItDown(mlm_client=client, mlm_model="gpt-4o-mini")
except Exception:
    pass
return MarkItDown()  # Fallback to standard MarkItDown
```

Reasons for fallback:
- Invalid API key
- OpenAI package not installed
- Network issues
- API quota exceeded

### Batch Image Processing

Process multiple images with enhanced OCR:

```bash
curl -X POST http://localhost:8000/upload \
  -F "files=@page1.jpg" \
  -F "files=@page2.jpg" \
  -F "files=@page3.jpg" \
  -o scanned_document.zip
```

Each image will be processed with GPT-4 Vision for accurate text extraction.

### Mixed Content Conversion

OpenAI only affects image processing. Other file types use standard conversion:

```bash
curl -X POST http://localhost:8000/upload \
  -F "files=@document.docx" \
  -F "files=@chart.png" \
  -F "files=@data.xlsx" \
  -o mixed_content.zip
```

- `document.docx` → Standard DOCX conversion
- `chart.png` → **OpenAI-enhanced** image OCR
- `data.xlsx` → Standard Excel table conversion

## Supported Image Formats

When OpenAI is enabled, these image formats benefit from enhanced OCR:

- PNG (`.png`)
- JPEG (`.jpg`, `.jpeg`)
- GIF (`.gif`)
- BMP (`.bmp`)
- TIFF (`.tiff`, `.tif`)
- WebP (`.webp`)

<Warning>
The MarkItDown library determines which files are processed as images. The API does not filter or validate image formats directly.
</Warning>

## Cost Considerations

### Pricing Model

OpenAI charges based on:
- **Input tokens**: Image size and complexity
- **Output tokens**: Generated text description
- **Model**: GPT-4o-mini rates

<Accordion title="Estimating costs">
  **Image processing costs** (approximate):
  - Small image (500x500): ~$0.001 - $0.003 per image
  - Medium image (1920x1080): ~$0.005 - $0.010 per image
  - Large image (4K): ~$0.015 - $0.030 per image

  These are estimates. Check [OpenAI's pricing page](https://openai.com/pricing) for current rates.
</Accordion>

### Cost Control

**Strategies to manage costs**:

1. **Use selectively**: Only enable OpenAI when processing image-heavy documents
2. **Monitor usage**: Set up OpenAI usage alerts in your account
3. **Resize images**: Preprocess large images to reduce token costs
4. **Disable when unnecessary**: Remove `OPENAI_API_KEY` for non-image workloads

load_dotenv()

# CORS origins — comma-separated in env

## Integration in Service Layer

The service layer seamlessly integrates OpenAI:

```python
# app/services/markitdown_service.py:92-118
async def convert_files_to_markdown(files: list) -> Tuple[bytes, str, str]:
    converter = _get_converter()  # Automatically uses OpenAI if configured
    results: List[Tuple[str, str]] = []

    with tempfile.TemporaryDirectory() as tmp_dir:
        tmp = Path(tmp_dir)

        for upload in files:
            raw = await upload.read()
            tmp_file = tmp / upload.filename
            tmp_file.write_bytes(raw)

            if _is_excel(upload.filename):
                results.extend(_convert_excel(tmp_file, upload.filename))
            else:
                result = converter.convert(str(tmp_file))
                stem = Path(upload.filename).stem
                results.append((f"{stem}.md", result.text_content))

    # Return results (single file or ZIP)
    # ...
```

### Python Test Script

```python
import os
import requests

def test_image_conversion(image_path: str):
    """Test image conversion with and without OpenAI."""
    
    # Check if OpenAI is configured
    has_openai = os.getenv("OPENAI_API_KEY") is not None
    print(f"OpenAI enabled: {has_openai}")
    
    with open(image_path, "rb") as f:
        files = {"files": f}
        response = requests.post(
            "http://localhost:8000/upload",
            files=files
        )
    
    if response.ok:
        content = response.content.decode("utf-8")
        print("\nExtracted content:")
        print(content)
        
        # Analyze quality
        word_count = len(content.split())
        print(f"\nWord count: {word_count}")
        
        if has_openai and word_count < 10:
            print("Warning: Expected more detailed output with OpenAI enabled")
    else:
        print(f"Error: {response.json()}")

# Test
test_image_conversion("screenshot.png")
```

## Troubleshooting

<Accordion title="Images not using OpenAI">
  **Symptoms**: Image conversion output is minimal or generic.

  **Solutions**:
  1. Verify API key is set: `echo $OPENAI_API_KEY`
  2. Check for typos in environment variable name
  3. Restart the API server after setting the key
  4. Check server logs for OpenAI initialization errors
  5. Verify API key is valid and has credits
</Accordion>

<Accordion title="OpenAI quota exceeded">
  **Symptoms**: Conversions fail or return errors after several requests.

  **Solutions**:
  1. Check your OpenAI account usage and billing
  2. Set up usage limits in OpenAI dashboard
  3. Temporarily disable OpenAI: `unset OPENAI_API_KEY`
  4. Consider upgrading your OpenAI plan
</Accordion>

<Accordion title="Slow conversion with images">
  **Symptoms**: Image conversions take significantly longer than other files.

  **Solutions**:
  1. This is expected - GPT-4 Vision processing takes time
  2. Typical response: 2-10 seconds per image
  3. Consider resizing very large images before upload
  4. For batch processing, implement queue system
  5. Disable OpenAI for time-critical operations
</Accordion>

<Accordion title="Import error: openai package">
  **Symptoms**: Server fails to start or logs show import errors.

  **Solutions**:
  ```bash
  # Install OpenAI package
  pip install openai
  
  # Or use requirements.txt
  pip install -r requirements.txt
  ```
  
  If you don't need OpenAI, the API will work fine without the package installed.
</Accordion>
